# Validation Checklist

Usar esta checklist antes de crear una issue.

- [ ] No hay issue duplicada activa equivalente.
- [ ] Título accionable y normalizado.
- [ ] Prioridad P0..P5 definida.
- [ ] Feature/Epic definida.
- [ ] Tipo story/task/subtask definido.
- [ ] Estado inicial válido (backlog|ready|in-progress).
- [ ] Owner real definido.
- [ ] Impacto definido.
- [ ] Dependencias explícitas (IDs o none).
- [ ] Entregable mínimo definido.
- [ ] Evidencia esperada definida.
- [ ] Criterios de aceptación definidos.
- [ ] Labels mínimas preparadas (prioridad, estado, feature/track, owner si aplica).

Resultado:
- Si todas en OK -> crear issue.
- Si alguna falla -> devolver error de validación y faltantes.
